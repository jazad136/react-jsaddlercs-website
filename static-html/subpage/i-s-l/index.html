Selected Algorithms

Insertion Sort I-S(L)

In SENG 2000, we gave demonstrations live of the components of this algorithm in the Java and Python language. We demonstrated our own flavor of the Some elements of memory while maintaining the workmanship of the algorithm.

Visualizing the Problem

Students were given visualization of the insertion sort solution to sorting. A player holds in their left hand a deck of unsorted "cards" with ordered symbols. In this case, the player's task is simply to move cards one at a time to the right place, searching from the leftmost spot of their left hand, for the card to go next into the right, while maintaining sorted order in the right hand.

Assignments

Students were assigned instances of the problem to fill in missing configurations of the algorithm transforming L into a sorted array. Before completing this assignment, we went through numbered exercises, that left parts of the problem of filling "L" half filled, so that students could ask in-class questions and experiment with their intended answer.

Runtime Analysis

The insertion sort algorithm was stepped through in detail to portray its runtime progression given certain inputs. 

Operations such as copying the current key item to its proper place in the output array can be found in lines 4 - 6 of our interpretation, while operations on selecting each element in the input for this procedure span lines 2 - 8. The output is a sorted arrangement starting from the element that is least in the leftmost spot, with "L" serving as the algorithm's output list by the end.

The whole procedure

adapted from Cormen [1]
Begin Algorithm I-S(L):
1. for j ← 2 ... length(L): 
2. |    key ← L [ j ] 
3. |    i ← j – 1 
4. |    while  i > 0 and L [ i ] > key : 
5. |    |    store-in L [ i + 1 ] ← L [ i ]
6. |    |    i ← i – 1
7. |    endw
8. |    store in L [ i + 1 ] ← key 
9. rof
End.

[1] Cormen, T., Leisersen C., Rivest R., and Stein C. Introduction to Algorithms. 3rd. ed. (2009) MIT Press.
