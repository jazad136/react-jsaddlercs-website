Selected Algorithms

Q-S(L, s, z)

In SENG 2000, we purposefully renamed some elements of memory while maintaining the workmanship of the algorithm, and I demonstrated working code following in Java to help students really understand the flow of its two major component subfunctions, "renamed" P-L(L,s,z) for "Partition List" and Q-S(L,s,z) which each do two different jobs. We take the definition from CLRS[1] and mix it up quite a bit!

Grand Example

Problems were defined in terms of our indexed lists.

We had one grand example given in class of a negative/positive input, that had an easy-to-grasp output of sorted integers from -5 to +5.

Partition

Students learned how the main memory elements used in partitioning point to spots in the "in-place" input that help move units left and right of the key variable of P-S, the partition element.

In our version of Quicksort, the partition variable is "returned" from the subprocedure to help the Quicksort operation complete the call in its entirety.

Quicksort Completes

The partitioning procedure P-L(L,s,z) is used to section off parts of the input less and greater than a certain element, return the partition element and, leave the input array in a certain state. The Q-S procedure can then use the new state of L to determine how to proceed. When the partition element returned happens to be the "median" of L, Quicksort can run optimally

Students were shown through demonstration Java code how to use the two "outputs" (the sorted array and returned partition element) of P-L to learn how Quicksort may continue to completely sort through an array if allowed to continuously recursively divide and manage elements of the input.

The whole procedure

adapted from Cormen [1]
Begin Q-S(L, s, z) 
1. if s < z: 
2. |	q ← P-L	( L , s ,      z ) 
3. | 	Q-S     ( L , s ,      q – 1 ) 
4. | 	Q-S     ( L , q + 1,   z )  
End.
    
Begin P-L(L, s, z):
1. i ← s – 1 
2. for j ← s … z – 1 
3. |    if L[ j ] < L[ z ] 
4. |    | 	i ← i + 1
5. |    |	swap-in( A, i, j )
6. rof
7. swap-in( A, i + 1, z )
8. return i + 1
End.

[1] Cormen, T., Leisersen C., Rivest R., and Stein C. Introduction to Algorithms. 3rd. ed. (2009) MIT Press.
